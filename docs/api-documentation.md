# API Documentation

This document provides comprehensive documentation for the Aleo Oracle Notarization Backend API. The API enables secure data attestation using Intel SGX enclaves and provides random number generation with cryptographic guarantees.

## Base URL

The base URL for the API is:
```
http://localhost:8000
```

The base URL for the metrics is:
```
http://localhost:8001
```

## Authentication

Currently, the API does not require authentication. All endpoints are publicly accessible.


## Error Response Format

All errors follow a consistent JSON format:

```json
{
  "errorCode": 1001,
  "errorMessage": "validation error: url is required",
  "errorDetails": "URL field was empty",
  "responseStatusCode": 400,
  "requestId": "req-123"
}
```

## Endpoints

### 1. Generate Attestation Report

**Endpoint:** `POST /notarize`

**Description:** Generates an SGX attestation report for data fetched from a specified URL. This is the main endpoint for creating verifiable attestations of external data.

**Request Body:**

```json
{
	"url": "https://archive-api.open-meteo.com/v1/archive?latitude=42.93869&longitude=-74.18819&start_date=2024-02-28&end_date=2024-02-28&daily=rain_sum",
	"requestMethod": "GET",
	"responseFormat": "json",
	"htmlResultType": "value",
	"selector": "daily.rain_sum.[0]",
	"requestHeaders": {
		"Accept": "*/*",
		"DNT": "1",
		"Upgrade-Insecure-Requests": "1",
		"User-Agent": "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/118.0.0.0 Safari/537.36"
	},
	"encodingOptions": {
		"value": "float",
		"precision": 5
	}
}
```

**Request Fields:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `url` | string | ✅ | Target URL to fetch data from |
| `requestMethod` | string | ✅ | HTTP method (GET/POST) |
| `selector` | string | ✅ | XPath or JSON key path for data extraction |
| `responseFormat` | string | ✅ | Response format (html/json) |
| `htmlResultType` | string | Conditional | Required for html format (element/value) |
| `requestBody` | string | Conditional | Required for POST requests |
| `requestContentType` | string | Conditional | Content type for POST requests |
| `requestHeaders` | object | ❌ | Additional HTTP headers |
| `encodingOptions` | object | ✅ | Data encoding configuration |
| `debugRequest` | boolean | ❌ | Enable debug mode (default: false) |

**Encoding Options:**

```json
{
  "value": "string|int|float",
  "precision": 0
}
```

## Usage Examples

### Example 1: Attest Bitcoin Price

```bash
curl -X POST http://localhost:8000/notarize \
  -H "Content-Type: application/json" \
  -d '{
	"url": "price_feed: btc",
	"requestMethod": "GET",
	"selector": "weightedAvgPrice",
	"responseFormat": "json",
	"requestHeaders": {
		"Accept": "*/*",
		"DNT": "1",
		"Upgrade-Insecure-Requests": "1",
		"User-Agent": "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/118.0.0.0 Safari/537.36"
	},
	"encodingOptions": {
		"value": "float",
		"precision": 6
	}
}'
```

## Data Extraction Selectors

### HTML Selectors

For `responseFormat: "html"`, use CSS selectors:

```json
{
  "selector": "/html/head/title",
  "htmlResultType": "value"
}
```

**Supported `htmlResultType` values:**
- `"element"` - Returns the HTML element
- `"value"` - Returns the text content

### JSON Selectors

For `responseFormat: "json"`, use dot notation:

```json
{
  "selector": "data.price.usd"
}
```

## Encoding Options

### Supported Data Types

| Type | Description | Example |
|------|-------------|---------|
| `string` | Text data | `"Hello World"` |
| `int` | Integer | `42` |
| `float` | Decimal number | `123.45` |

```json
{
  "value": "float",
  "precision": 2  // Rounds to 2 decimal places
}
```

## Error Handling

### Common Error Scenarios

1. **Validation Errors (1000-1999)**
   - Missing required fields
   - Invalid data formats
   - Domain not whitelisted

2. **Enclave Errors (2000-2999)**
   - SGX hardware issues
   - Quote generation failures

3. **Data Extraction Errors (4000-4999)**
   - Network timeouts
   - Selector not found
   - Invalid response format

4. **Exchange Errors (6000-6999)**
   - API rate limits
   - Exchange unavailable
   - Invalid symbol

### Error Response Example

```json
{
  "errorCode": 4001,
  "errorMessage": "data extraction error: invalid http request",
  "errorDetails": "Connection timeout after 30 seconds",
  "responseStatusCode": 500,
  "requestId": "req-abc123"
}
```

## Security Considerations

### Domain Whitelisting

Only whitelisted domains can be attested. Check `/whitelist` endpoint for allowed domains.

### SGX Attestation

- All attestations are cryptographically signed
- Enclave measurement (MRENCLAVE) ensures code integrity
- Attestation reports can be verified independently

### Data Privacy

- Request data is processed within SGX enclave
- No sensitive data is logged or stored
- All communications use HTTPS

## Troubleshooting

### Common Issues

1. **Domain Not Whitelisted**
   - Check `/whitelist` endpoint
   - Contact administrators to add domain

2. **SGX Quote Generation Fails**
   - Verify SGX hardware is available
   - Check enclave configuration
   - Review system logs

3. **Data Extraction Fails**
   - Verify selector is correct
   - Check target website availability
   - Review network connectivity

4. **Invalid Request Format**
   - Validate JSON structure
   - Check required fields
   - Verify data types
